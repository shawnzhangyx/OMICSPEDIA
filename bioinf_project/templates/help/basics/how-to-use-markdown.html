How do I format the wiki and posts using Markdown?

Omicspedia uses Markdown for most of the text contents, except for comments. If you're new to Markdown, you may want to visit John Gruber's offical Markdown webiste for syntax reference.

here are some example syntax uses and an example page:

Headers

We recommend using atx-style headers (use 1-6 hash characters at the start of the line, corresponding to header levels 1-6). For example: 

# This is an H1
## This is an H2
### This is an H3

Blockquotes

Markdown uses e-mail style > character for blockquoting. Simply put a > before every line. 
 
> This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet,
> consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
> Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.
> 
> Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse
> id sem consectetuer libero luctus adipiscing.
Blockquotes can be nested by adding additional level of >: 
> This is the first level of quoting.
>
> > This is nested blockquote.
>
> Back to the first level.
Blockquotes can contain other Markdown elements, such as headers, lists and code blocks.

Lists

Unordered lists uses asterisks, pluses and hyphens --interchangably -- as list markers: 
*   Red
*   Green
*   Blue
Ordered lists use numbers followed by periods: 
1.   Red
2.   Green
3.   Blue

Code Blocks

To produce a code block, leave 4 spaces before each line of the code. For example: 
This is a regular paragraph. 
    this is code block 
    code block continued
You can also create a code block by including the code block within more than 4 ~ symbols. You can also specify the programing language. 
~~~~~~~{.python}
def funtion(var1, var2):
    sum = var1+var2
~~~~~~~

Inline Code Blocks

To produce an inline code block, include the code within two ` symbols. 
 This is an `inline code block`.

Horizontal Rules

You can produce a horizontal rule tag (<hr />) by placing three or more hyphens, asterisks, or underscores on a line by themselves. 
***
_______

-----------------

Links

Markdown supports two styles of links: inline and reference. To create an inline link, use a set of regular parenthesis immediately after the link's closing square bracket. Inside the parenthesis, put the URL and an optional title for the link. 
 This is [an example](http://example.com/ "Title") inline link.
[This link](http://example.net/) has no title attribute.
To create a reference-style link, use a square bracket instead of parenthesis, and put ID inside the square bracket. Then, anywhere in the text, you can define the link ID in a sparate line: 
 This is an [example][id] reference-style link. 
...
[id]: http://example.com/ 'optional title'

Images

The image syntax is similar to the syntax of links. It supports two styles of links: inline and reference. The inline syntax looks like the following: 
 ![alt text](URL/image.jpg)
![alt text](/path/to/image.jpg "optional title")
It starts with an exclamation mark, followed by alternative title enclosed with square brackets. And then followed by a set of parentheses containg the URL or path to the image, and an optional title. Reference style image is like this: 
 ![alt text][id]
[id]: URL/image.jpg  "optional title"
Currently Markdown has no support for specifying the dimension or position of the image. But we're working to implementing that for this website.

Wikilinks

To produce links to wiki phrases, simply put two square brackets around the wiki phrases. 
 [[wiki words]]

Table of Content

To produce a table of content, put the following line anywhere you want the table of content to be. 
 [TOC]

Footnote/References

To produce a footnote or reference, put a footnote ID inside square brackets and insert the content anywhere in the text. 
 Footnotes[^1] have a label[^@#$%] and the footnote's content.

[^1]: This is a footnote content.
[^@#$%]: A footnote on the label: "@#$%".
